GregTech is notable for having some of the longest and most detailed configuration files of any mod. Its config files are so long and detailed that there are actually three of them, and they have their own folder (minecraft/config/GregTech/). As such, each configuration file will receive its own page.
The three config files:
1. BlockItemIDs.cfg
This one is pretty much exactly what its name says. If you have a block or item ID conflict that throws a crash, this is where you could fix it on the GregTech end.
2. GregTech.cfg
This file contains the settings core to GregTech, those that are not affected by other mods.
3. DynamicConfig.cfg
This is the really big one. DynamicConfig contains configuration options for every single recipe GregTech adds. It is called DynamicConfig because it changes itself to reflect your mod list, as GregTech adds many recipes to itself based on what other mods you have.


Note: Because of Greg's tendency to change things under the hood, every time GregTech updates, you may need to re-generate one or more of your config files. If you change a line in the config files and it does nothing, most likely that config line is deprecated, and you will need to re-generate that file. Keep backup configs.
