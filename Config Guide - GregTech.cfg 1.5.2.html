GregTech.cfg is the base configuration file for GregTech. This file contains settings affecting some of the more core functions of GregTech, those that are not influenced by other mods. Yes, this does contain several of his nerfs.
The following is a freshly generated copy+paste of GregTech.cfg, from a Minecraft 1.5.2 instance running GregTech 3.11.
  • All lines with bullet points, like this line, are comments added to clarify the lines in the configuration file. They are not part of the acutal file.
Many lines in this file are related to one another. Comments will be added after sets of related lines.

#compatibility
####################
compatibility {
B:Buildcraft.Energy=true
B:Industrialcraft.Energy=true
B:UniversalElectricity.Energy=true
  • GregTech machinery can receive energy from IndustrialCraft2, Buildcraft, and Universal Electricity energy networks. If you are encountering compatibility issues, or if for balancing reasons you don't want your GregTech machines to receive power from one or more other energy systems, you can disable those interactions from here.
  • Remember that GregTech machines will not output any form of energy other than EU.
}

#features

features {
I:DigitalChestMaxItemCount=32768
I:MatterFabricationRate=16666666
  • This is the amount of EU required for a Matter Fabricator to produce 1 UU Matter. It is based on (but independent from) the IC2 Mass Fabricator's UU Matter cost, with Scrap applied (because the Matter Fabricator requires Scrap or a similar amplifier).
I:MaxLogStackSize=64
I:MaxOreStackSize=64
I:MaxOtherBlockStackSize=64
I:MaxPlankStackSize=64
  • GregTech includes the option to reduce the stack size of various items, as a means of adding a little extra difficulty and realism to your mining runs. These values do not affect non-block items, and cannot be set lower than 16. If set to lower than 16, they will be read as 16.
I:QuantumChestMaxItemCount=2000000000
  • The Digital Chest and Quantum Chest are industrially scaled-up versions of Factorization's Barrels. They store a very large amount of 1 item, and once an item has been inserted into them they will not accept any other items. Unlike Barrels, however, if the last item is removed from them, they will remember their internal filter, and still only accept that one item. If they are disassembled and re-placed, their internal filter will be reset.
  • Due to integer overflows, it may not be possible to set the Quantum Chest to have a higher capacity. Not that you would ever need to, as the default configuration value is 2 BILLION.
B:Reactorplanner=true
B:Seedscanner=true
  • Reactor Planner and Seed Scanner are two functions of the GregTech Computer Cube. Set to false to disable those functions.
  • Reactor Planner allows you to lay out the design of a Nuclear Reactor and simulate it in a controlled, safe environment. So, if you have a reactor design that would blow up your base, you can find out about it before you build the reactor and blow up your base.
  • Seed Scanner allows you to place a Seed Bag (produced by an IC2 crop) into the Computer Cube, and scan it for its attributes.
I:TincellsPer4Tin=4
  • When it was originally implemented, this config line would simply change the output stack size of the conventional IC2 crafting recipe for Empty Cells. (Originally, 4 Tin Ingots in a diamond shape -> 16 Empty Cells.) It now does several more things.
  • It changes the stack size output of the Empty Cell crafting recipe (4 Tin Plates in a diamond)
  • It changes the Empty Cell crafting recipe in the Assembling Machine. By default, the Assembling Machine recipe is 1 Tin Ingot = 1 Empty Cell, however if this line is changed, the Assembling Machine recipe will change to fit the ratio set here. (For example, if you set I:TincellsPer4Tin=7, the Assembling Machine recipe would change to 4 Tin Ingots = 7 Empty Cells.) It does try to reduce the ratio to its lowest terms (hence the default 1=1 recipe). (For example, if you set I: TincellsPer4Tin=8, the Assembling Machine recipe would change to 1 Tin Ingot = 2 Empty Cells.)
  • Note that the differing ratios do not affect the time or energy cost of the Assembling Machine recipe.
  • Additionally, when Centrifuging Lava in an Industrial Centrifuge, the Tin produced will be equal to 2 + (amount of tin spent in containers). By default (4 cells per 4 tin), Centrifuging 16 Lava Cells will produce 18 Tin Ingots. If set to 8 cells per 4 tin, Centrifuging 16 Laval Cells will produce 10 Tin Ingots.
I:UpgradeStacksize=4
  • A nerf to the idea of dropping 10 Overclocker Upgrades on a vanilla IC2 Macerator.
}

#general
####################
general {
B:ConnectedMachineCasingTextures=true
  • Like how Glass blocks placed next to each other will connect to form a window, Machine Casings placed next to each other will connect to form a large machine hull.
  • Different types of Machine Casings (Standard, Reinforced, Advanced) will not connect.
  • This is a client-side rendering option, and will have no effect on a server.
B:Debug=false
B:Debug2=false
  • Adds a whole bunch more lines to the forge and gregtech log files. Not useful to the general player.
B:ShowCapes=true
  • Whether or not to render GregTech Intergalactical capes. This is a client-side rendering option, and will have no effect on a server.
B:TooEasyMode=false
  • A man who shall not be named spent a little too much time harassing Greg over his Forestry Bronze recipe nerf, and his removal of the Tin Bucket recipe. The result was this. TooEasyMode un-nerfs the conventional Bronze recipe (shapelessly craft 3 Copper ingots and 1 Tin ingot), but changes it to produce 111 Bronze Ingots. Additionally, a shapeless crafting recipe is added that turns 1 Dirt into 1 UU Matter, and the Tin Bucket recipe has been re-added (and produces 111 Buckets).
  • Don't use this.
  • Seriously, don't.
B:WoodNeedsSawForCrafting=true
  • Probably the most controversial nerf Greg has done, added in version 3.10a, the vanilla crafting recipe of 1 Wood Log -> 4 Wood Planks was nerfed to produce 2 Wood Planks, and a line of Saws were added. Crafting 1 Wood Log with a Saw above it will yield the conventional 4 Planks. Additionally, the 2 Planks -> 4 Sticks recipe has been nerfed, producing 2 Sticks. Crafting 2 Planks with a Saw above them will yield the conventional 4 Sticks. Each of these recipes will consume 1 durability point from the Saw.
  • When this config is set to false, the vanilla recipes are not affected, but an additional set of recipes are added, and crafting Wood with a Saw will cause it to yield 5 Planks in stead of 4, and crafting Planks will a Saw will yield 5 Sticks in stead of 4.
  • This is the nerf (and the completely ignored config line) that caused that marvelous burst of outrage and code war between GregoriusT, mDiyo, the Feed The Beast forums, and the IndustrialCraft2 forums. It also created a wonderful analogy in which Minecraft modding was compared to a hot dog with ketchup, and a hot dog with sulfuric acid. There were about 6 different interpretations of that analogy.
  • For more information on Saws, see the Tools page.
B:disable_STDERR=false
B:disable_STDOUT=false
  • If you have a mod that's generating a ton of error messages and spamming your logs, you can force Forge to not print out STDERR or STDOUT messages. This was implemented due to a bug that arose some time in April, wherein some mod was generating a 10-line error message stack trace every tick, and Forge client and server logs were expanding to hundreds of megabytes of text.
B:harderstone=false
  • For that little extra bit of realism, this significantly increases the Hardness of vanilla Stone and Cobblestone, causing them to take much longer to mine.
  • Normally, Stone has a Harness of 1.5, Cobblestone has a Hardness of 2.0, and both have a Blast Resistance of 6.0.
  • With this enabled, Stone has a hardness of 16.0, Cobblestone has a Hardness of 12.0, and Stone Bricks have a Hardness of 24.0. Their Blast Resistances are equal to their new Hardness values.
  • Netherrack, Mossy Cobblestone, Sandstone, and End Stone are not affected by this config.
  • Stairs, Slabs, and Walls made of Stone are not affected.
  • Stone added by other mods (such as Underground Biomes) is not affected.
B:hiddenores=true
  • GregTech Ores have a chance to camouflage themselves underground by stealing the texture of an adjacent Stone, Netherrack, or End Stone block. This has an added effect; if an Ore has no Stone blocks adjacent to itself, it will in stead assume a very obvious and colorful "vein core" texture.
  • A lot of people have found this to be very annoying, which is why this config line exists, and why SirusKing drew up a set of alternate textures.
  • For more information, see the Ores page.
  • This is a client-side rendering option (as the ore camouflage effect is done in client-side rendering), and will have no effect on a server.
B:online=true
  • GregTech has a dropbox that it polls whenever Minecraft starts, to check the GT cape list among other things. This is not an essential check, and if online=false will not be done. Useful for if you know you will be playing offline, and you don't want to see an error message.
B:smallerStoneToolDurability=true
B:smallerWoodToolDurability=true
  • An additional pair of nerfs in GT 3.10a, these two configs add an extra incentive to get out of the stone age.
  • smallerWoodToolDurability sets all Wooden tools to have a durability of 13 (from the vanilla 60)
  • smallerStoneToolDurability sets all Stone tools to have a durability of 49 (from the vanilla 132)
B:unsuspiciousHungerConfig=true
  • Minecraft "Survival" mode isn't all that hard to survive in. This config does two things:
  • It increases the rate at which food points are consumed passively
  • It adds "hunger ticks," large ticks that occur about every 5 minutes that drain a large amount of hunger, based upon how many items the player is carrying in their inventory. These hunger ticks have been known to drain a player's entire food bar.
}

#machines
####################
  • This set of configurations are a reminder of why one must be very careful when building a factory out of industrial-grade machinery. Probably the biggest rule is, "don't build a factory outside."
machines {
B:constant_need_of_energy=true
  • If a machine is not receiving the full EU/t required to run, it will stop producing, lose its production, and reset. Consequently, if you screw up your wiring or your energy feeds, or your Batbox dies, you can waste a lot of energy and get nothing done.
  • If set to false, if a machine runs out of energy, it will pause its production, as a vanilla IC2 machine would.
B:explosions_on_nonwrenching=true
  • If a machine is broken with a Pickaxe or destroyed by an explosion, the machine will explode rather than dropping. If this is set to false, a non-wrenched machine will drop as a Machine Parts item.
B:fire_causes_explosions=true
  • If a machine catches fire, it has a chance to explode.
B:lightning_causes_explosions=true
  • If a machine is struck by lightning, it will explode.
B:machines_flammable=true
  • Machines can catch and carry a fire. (Like wood.)
B:rain_causes_explosions=true
  • Machines in rain storms have a chance to catch fire and/or explode. In-game, this can easily be prevented by putting any block above the machine, or by putting a Cover on the top of the machine block.
B:wirefire_on_explosion=true
  • This is the really dangerous one. When a GregTech machine explodes, it will emit energy from all 6 of its sides. It will emit one packet each of, in order, 32EU, 128EU, 512EU, 2048EU, and 8192EU. This will fry any cables short of Superconductor Wires, and overload and explode any machine not capable of accepting 8192EU/p (Quad-EV).
}

#unificatortargets
####################
unificatortargets {
B:Forestry=false
B:Railcraft=false
B:Redpower=true
B:Thaumcraft=true
B:ThermalExpansion=false
  • You know how many mods add some form of Copper? It's not uncommon to find 4 or 5 or 6 different Copper Ores underground. If each of those was treated as a separate substance, nobody would be able to get any of their crafting recipes right because everyone would be hunting for which mod added which form of copper for which recipe. This is where the Forge OreDictionary comes into play, allowing your 5 different Copper Ingots to be treated as the same substance, despite being added by 5 different mods and having 5 different textures. However, those 5 different Copper Ingots are still different items, and as such, do not stack. This is where GregTech OreDictionary Unification comes into play. With this function, whenever a recipe would produce a Copper Ingot, GregTech overwrites that recipe to cause it to produce the copper added by only one specific mod. This config block allows you to choose what mod takes precedence; which mod's copper is produced. Note that this does not affect dungeon loot.
  • Additionally, the GregTech Electric Crafting Table contains a mode called Unification Mode. If set (the option looks like a DNA molecule with an arrow along the bottom), the Electric Crafting Table will check any OreDictionary-registered items with the Unification settings, and if they don't match (for example, a Thermal Expansion Copper Ingot, while B:ThermalExpansion=false), it will "Unificate" them to make them complaint with the Unification settings. For anti-exploit reasons, this function only works for metals and ores.
  • By default, Redpower and Thaumcraft are set to true; Redpower to allow for making Microblocks out of metal storage blocks, and Thaumcraft because the Nuggets produced by its Infernal Furnace are not otherwise Unificated automatically, and we still want them to stack correctly.
  • It is highly recommended that you select either Forestry, Railcraft, or Thermal Expansion, or leave them all to false. If more than one is set to true, it may lead to inconsistencies across different forms of recipes.
  • If all are set to false, the GregTech versions of these materials will be produced.
  • This config block usually works best if you just don't touch it.
}
